# Проектирование: Agent Sim — 2D мир с AI-агентами

## Концепция

2D-мир на Phaser.js, где живут AI-агенты (10-20 персонажей). 
Каждый агент — LLM-персонаж со своим характером, расписанием, целями.
Игрок может свободно перемещаться по миру, наблюдать за агентами, 
подглядывать их мысли/статус. Агенты взаимодействуют друг с другом через LLM.

Как Agentopia (habr.com/ru/articles/1047352/), но в 2D, браузере, и легковеснее.

### 2. Технический стек

- **Движок:** Phaser.js 3 (через npm/vite)
- **Фронт-сборка:** Vite + TypeScript
- **LLM:** OpenRouter API (GPT-4o-mini для агентов, дёшево)
- **Хранение:** IndexedDB (для состояний агентов, логов)
- **Карта:** Tiled (JSON-карты) или программная генерация

### 3. Архитектура бэкенда (Go или Python)

**Go** (как в B5 проекте) — один бинарник, embed статики, фамильярно.
**Python** — если numpy/scipy для обработки логов, анализа поведения.

Выбрать и обосновать.

### 4. Мир и карта

- Карта в тайлах (Tiled editor): комнаты, коридоры, проходы
- Каждая зона имеет название, описание для LLM-контекста
- Агенты перемещаются по карте (pathfinding Phaser или простой A*)
- Размер карты: ~20×30 тайлов (небольшая, для 10-20 агентов)

### 5. Агенты

Структура агента:
- **ID, имя, роль** ("официант", "менеджер", "посол")
- **Характеристики:** charisma, industriousness, sociability, emotional_stability (1-10)
- **Текущее состояние:** location, activity, mood, energy
- **Память:** Last 10 significant events (для контекста LLM)
- **Расписание:** цикл действий (work, eat, rest, socialize)
- **Отношения:** Map<agentID, disposition> (как агент относится к другим)
- **Долгосрочные цели:** "стать самым популярным", "накопить деньги", "найти любовь"

### 6. Система тиков (цикл симуляции)

- Тик = 1 секунда реального времени = 5 минут игрового времени
- День = 288 тиков (~5 минут реального времени)
- Каждый тик:
  1. Обновление позиций агентов (движение к цели)
  2. Проверка встреч (два агента в одной зоне → возможно взаимодействие)
  3. LLM-запрос (не каждый тик, а по событиям: встреча, смена активности, решение)
  4. Обновление состояния (mood, energy, relationships)

### 7. LLM-интеграция

- Агенты НЕ делают LLM-запрос каждый тик (дорого)
- Триггеры для LLM:
  - Встреча с другим агентом (генерация диалога + изменение отношений)
  - Смена активности (выбор: чем заняться?)
  - Конец дня (рефлексия: что запомнить?)
  - Кризис (energy < 20%, mood < 20%)
- **System prompt** для агента: описание мира, его роль, характер, текущее состояние
- **Контекст:** последние 5 событий, отношения с другими агентами
- **Формат ответа JSON:** { action, dialogue?, mood_change, relationships_change }
- **Кеш:** повторяющиеся ситуации не перегенерять

### 8. UI

- **2D мир** (Phaser): камера следует за игроком, тайловые полы, стены, объекты
- **HUD:** имя локации, список агентов рядом, часы (игровое время)
- **Клик по агенту:** панель агента (имя, настроение, статус, мысли, отношения)
- **Режим наблюдения:** камера следует за выбранным агентом
- **Лог событий:** список "что произошло" (в стиле: "Мария поговорила с Джоном. Джон рассержен.")
- **Панель статистики:** графики/числа: среднее настроение, количество взаимодействий

### 9. План реализации (фазы)

- Фаза 0: Каркас (Vite + Phaser + камера, минимальная карта)
- Фаза 1: Агенты (движение, расписание, базовое состояние)
- Фаза 2: LLM-агенты (интеграция, диалоги, память)
- Фаза 3: UI (панель агента, лог, статистика)
- Фаза 4: Контент (10-20 агентов, Tiled-карта, сбалансированные расписания)
- Фаза 5: Игрок-наблюдатель (свободная камера, выбор агента, вмешательство?)
- Фаза 6: Анализ (построение графов отношений, дневники агентов)

## ВАЖНО

- 10-20 агентов, не 100
- LLM-запросы только по триггерам, не в каждый тик
- Дёшево: GPT-4o-mini, кеш
- Один разработчик в свободное время
- Браузерное SPA, без сервера (весь Go-бэкенд опционален)
- Можно использовать наработки из b5-game (Go-сервер, LLM-клиент) если выбран Go
